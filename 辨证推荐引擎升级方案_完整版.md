## 一、升级背景

### 1.1 现有系统概况

- **技术栈**：Vue 3 + Vite + Pinia
- **当前问卷**：14题（Q1-Q14）
- **推荐逻辑**：`answerToSymptomMap` 关键词命中 → 匹配药品 `fit_symptom` 字段 → 按命中数量排序
- **存在问题**（豆包评审指出）：
  1. 推荐逻辑是「有/无」二元命中，无权重区分，精准度不足
  2. 发热只有单一阈值（38.5℃），无分级预警
  3. 推荐结果没有「为什么推荐这个」的文字说明
  4. 缺少正向联用建议规则（只有配伍禁忌）
  5. Q3 出汗题经方含义未向宝妈解释清楚

### 1.2 升级目标

| 目标 | 说明 |
|------|------|
| 提高推荐精准度 | 引入权重评分机制，替代二元命中 |
| 提高安全性 | 发热三级分级 + 危险信号检测 + 证型互斥拦截 |
| 提高可解释性 | 每条推荐给出「推荐理由」文字 |
| 补充中医辨证深度 | 落实经方辨证条款（麻黄汤/桂枝汤/少阳证等）+ 补全遗漏证型 |
| 补充联用指导 | 增加正向联用建议（非禁忌），补全高频联用场景 |

## 三、权重评分机制（核心算法升级）【豆包终审已确认】

### 3.1 设计原则

1. **保留原有 `answerToSymptomMap` 不变**（向后兼容，不删除）
2. **新增 `syndromeWeightConfig` 配置对象**，定义证型加权规则
3. **新增 `syndromeExclusionRules` 配置**，定义证型互斥拦截规则
4. **新增 `compoundPriorityConfig` 配置**，复合证型优先于单一证型
5. **评分逻辑在 `ResultView.vue` 的 `matchedMedicines` 计算属性中实现**
6. **评分结果同时记录「命中详情」**，用于生成推荐理由文字

### 3.2 权重规则设计（豆包终审调整梯度）

```
总分 = Σ(症状命中分) + Σ(证型加权分) + Σ(年龄罚分) + Σ(哮喘史加权分)
然后经过：证型互斥过滤 → 复合证型优先级排序

① 症状命中分（每命中1个关键词 +1分）：
   - answerToSymptomMap 映射的每条关键词，在药品 fit_symptom 字段中命中 → +1分
   - 同一题多个关键词命中 → 累加
     例：Q3 选了 no_sweat_cold，映射7个词，命中数量越多分越高

② 证型加权分（豆包终审调整梯度，拉大层级差距）：
   判定为「湿热/暑湿感冒」   → 湿热类药品 +8分（最高优先级，最易误治）
   判定为「风寒挟积食」       → 风寒+消食类药品 +6分（新增证型，高频场景）
   判定为「积食挟风热」       → 风热+消食类药品 +6分
   判定为「少阳证（寒热往来）」 → 小柴胡类药品 +6分
   判定为「流感/病毒性感冒」   → 抗病毒类药品 +6分
   判定为「风热感冒」          → 风热类药品 +4分
   判定为「风寒感冒（无汗）」  → 风寒类药品 +4分
   判定为「风寒感冒（有汗）」  → 桂枝类药品 +4分
   判定为「痰热咳嗽」          → 清热化痰类药品 +4分
   判定为「久咳虚燥」          → 润肺止咳类药品 +2分

③ 年龄罚分（安全优先，一票否决）：
   0-3岁：药品 baby_taboo 含「婴幼儿禁用」→ -100分（不显示）
   0-3岁：药品 baby_taboo 含「婴幼儿慎用」→ -5分（往后排，但不隐藏）

④ 哮喘史加权分（豆包终审 +3 → +6）：
   有哮喘史 + 咳嗽症状 
   → 药品 fit_symptom 含「平喘/麻黄/杏仁/气喘/哮喘/咳喘」→ +6分（专项高权重）
   注意：0-3岁已过滤，不再加分（麻黄类对婴幼儿有风险）
```

```javascript
// ============================================================
// 证型加权配置（权重评分机制用）
// 用于把用户答题结果映射为「证型命中」，给对应药品加权
// 【豆包终审】权重梯度已调整：8/6/4/2 四档
// ============================================================
export const syndromeWeightConfig = {
  // 键 = 对应 syndromeGroups 的 id
  // weight = 加权分数
  // matchLogic = 判定该证型命中的答题条件（函数，接收 answers 对象）
  // matchSymptoms = 向用户展示的「命中症状」中文描述（用于推荐理由）

  // ========== 最高优先级 8分档 ==========

  damp_heat: {
    weight: 8,   // 【豆包终审】5 → 8，最高优先级，湿热最易误治
    name: '湿热/暑湿感冒',

      const hasThickTongue = answers.tongue && answers.tongue.includes('tongue_thick')
      const hasStickyStool = answers.stool === 'sticky_stool'
      const hasDampSign = (answers.fever === 'low_fever' && answers.stool === 'diarrhea')
      return hasThickTongue && (hasStickyStool || hasDampSign)
    },
    matchSymptoms: ['舌苔厚腻','大便黏臭','暑湿季节']
  },

  // ========== 高优先级 6分档 ==========

  cold_food: {
    weight: 6,
    name: '风寒挟积食（寒积感冒）',
    matchLogic: (answers) => {
      // 风寒条件：怕冷 + (无汗 或 清涕)
      const isCold = answers.cold_heat === 'afraid_cold'
        && (['no_sweat_cold','no_sweat_nocold'].includes(answers.sweat)
            || answers.nose === 'clear_snot')
      // 积食条件：腹胀 或 食欲差 或 舌苔白厚
      const hasFood = answers.appetite
        && (answers.appetite.includes('bloat') || answers.appetite.includes('no_appetite'))
      // 舌苔厚腻也是积食信号
      const hasTongueFood = answers.tongue && answers.tongue.includes('tongue_thick')
      return isCold && (hasFood || hasTongueFood)
    },
    matchSymptoms: ['怕冷','腹胀','大便稀溏','舌苔白厚','无汗']
  },

  food_stag: {
    weight: 6,   // 【豆包终审】4 → 6
    name: '积食挟风热',
    matchLogic: (answers) => {
      const hasFood = answers.appetite 
        && (answers.appetite.includes('no_appetite') || answers.appetite.includes('bloat'))
      const hasHeat = answers.cold_heat === 'afraid_heat'
                        || answers.nose === 'yellow_runny'
                        || answers.cough === 'yellow_phlegm'
      return hasFood && hasHeat
    },
    matchSymptoms: ['积食','口臭','黄痰','便秘']
  },

  class_ic_shaoyang: {
    weight: 6,   // 【豆包终审】4 → 6
    name: '少阳证（寒热往来）',
    matchLogic: (answers) => {
      return answers.cold_heat === 'alternate'
    },
    matchSymptoms: ['寒热往来','口苦','食欲不振','恶心']
  },

  flu_viral: {
    weight: 6,   // 【豆包终审】4 → 6
    name: '流感/病毒性感冒',
    matchLogic: (answers) => {
      return answers.fever === 'high_fever'
          && (answers.throat && answers.throat.includes('sore_throat'))
    },
    matchSymptoms: ['高热','咽喉肿痛','全身症状重']
  },

  // ========== 中等优先级 4分档 ==========

  wind_heat: {
    weight: 4,   // 【豆包终审】3 → 4
    name: '风热感冒',
    matchLogic: (answers) => {
      return answers.cold_heat === 'afraid_heat'
          || answers.nose === 'yellow_runny'
          || (answers.throat && answers.throat.includes('sore_throat'))
    },
    matchSymptoms: ['怕热','黄鼻涕','咽喉肿痛']
  },

  wind_cold: {
    weight: 4,   // 【豆包终审】3 → 4
    name: '风寒感冒',
    matchLogic: (answers) => {
      return answers.cold_heat === 'afraid_cold'

    },
    matchSymptoms: ['怕冷','无汗','清鼻涕']
  },

  class_ic_cold_mahuang: {
    weight: 4,   // 【豆包终审】3 → 4
    name: '风寒表实（麻黄汤证）',
    matchLogic: (answers) => {
      return answers.sweat === 'no_sweat_cold'  // 不出汗+怕冷厉害

    matchSymptoms: ['无汗','怕冷','身痛']

  cough_heat: {
    weight: 4,   // 【豆包终审】3 → 4
```

### 3.4 评分算法伪代码（供审查逻辑正确性）

```
函数 calcMedicineScore(medicine, answers, ageGroup, allergyHistory) 返回 { score, hitSymptoms, matchedSyndromes }

  分数 = 0
  命中症状列表 = []
  命中证型列表 = []
  激活的证型集合 = []   // 所有 matchLogic 返回 true 的证型
  
  // ① 症状关键词命中分
  映射症状集 = 从 answerToSymptomMap 收集所有答案对应的关键词（扁平化数组）
  对 映射症状集 中每个词：
    如果 药品.fit_symptom 包含该词：
      分数 += 1
      把该词加入 命中症状列表
  
  // ② 证型加权分（先收集所有命中的证型）
  对 syndromeWeightConfig 中每个证型配置：
    如果 matchLogic(answers) 返回 true：
      把 证型key 加入 激活的证型集合
  
  // ②-1 证型互斥拦截
  对 syndromeExclusionRules 中每条规则：
    如果 规则.triggerSyndromes 和 激活的证型集合 有交集：
      从 激活的证型集合 中移除 规则.blockedSyndromes 内的所有证型
  
  // ②-2 加权（只对互斥过滤后仍然激活的证型）
  对 激活的证型集合 中每个证型key：
    配置 = syndromeWeightConfig[证型key]
    如果 药品.id 在对应 syndromeGroups[证型key].medIds 中：
      分数 += 配置.weight
      把 配置.name 加入 命中证型列表
  
  // ②-3 复合证型加成分
  如果 compoundPriorityConfig.enabled：
    如果 激活的证型集合 中同时包含某个 compoundPatterns 定义的多组证型：
      分数 += compoundPriorityConfig.compoundBonus  // +2分保证置顶
  
  // ③ 年龄罚分（安全优先）
  如果 ageGroup === '0-3'：
    如果 药品.baby_taboo 包含 '婴幼儿禁用'：
      分数 -= 100   // 一票否决，不显示
    否则如果 药品.baby_taboo 包含 '婴幼儿慎用'：
      分数 -= 5     // 往后排，但不隐藏
  
  // ④ 哮喘史/过敏史加权【豆包终审 +6】
  如果 allergyHistory 是 'asthma_history' 或 'allergy_history'：
    如果 ageGroup !== '0-3'：  // 0-3岁已过滤，不再加分
      如果 asthmaBoostConfig.boostKeywords 中任意词 在 药品.fit_symptom 中：
        分数 += asthmaBoostConfig.boostScore  // +6分
        把 '哮喘史加权' 加入 命中证型列表
  
  返回 { score, hitSymptoms: 命中症状列表, matchedSyndromes: 命中证型列表 }
```

### 3.5 排序规则（含互斥+复合优先级）

```
1. 分数 < 0 的药品 → 过滤掉（不显示）
2. 按分数从高到低排序
3. 分数相同的药品 → 按 (isPrimary ? 0 : 1) 排序（首选药排前面）
4. 最终结果：
   - 首选药：分数最高的1个
   - 备选药：分数 > 0 且不是首选药 的前 N 个（N = 3 或 4，见推荐数量限制）
5. 互斥规则确保：同一结果页不会同时出现风寒药和风热药作为首选
```

## 五、推荐理由文字展示【豆包终审确认完全合格】

### 5.1 设计原则

1. 每条推荐药品下方显示**一行灰色小字**（字号比正文小2号）
2. 理由来自**实际命中的症状和证型**，不是固定模板
3. 格式：「命中【症状A + 症状B + …】，辨证属于【证型C / 证型D】」
4. 没有命中任何症状时（罕见），显示：「综合辨证推荐，请结合实际情况判断」

### 5.2 数据来源

```javascript
  hitSymptoms: ['发热', '咽喉肿痛', '便秘', '高热'],

}
```

### 5.3 推荐理由生成函数

```javascript
// ============================================================
// 生成推荐理由文字
// ============================================================
function getRecommendReason(medicine, scoreDetail) {
  const parts = []

  // 命中症状
  if (scoreDetail.hitSymptoms && scoreDetail.hitSymptoms.length > 0) {
    const symptomsStr = scoreDetail.hitSymptoms.slice(0, 5).join(' + ')  // 最多显示5个
    parts.push(`命中【${symptomsStr}】`)
  }

  // 命中证型
  if (scoreDetail.matchedSyndromes && scoreDetail.matchedSyndromes.length > 0) {
    const syndromesStr = scoreDetail.matchedSyndromes.join(' / ')
    parts.push(`辨证属于【${syndromesStr}】`)
  }

  // 联用提示
  if (medicine.isCombined) {
    parts.push('（联合用药，增强疗效）')
  }

  //  fallback
  if (parts.length === 0) {
    return '综合辨证推荐，请结合实际情况判断'
  }

  return parts.join('，')
}
```

### 6.4 联用建议匹配函数

```javascript
// ============================================================
// 匹配联用建议
// 【豆包终审新增】ageGroup 参数用于年龄限制检查
// ============================================================
export function matchCombinationSuggestions(answers, ageGroup = null) {
  const matched = []

  for (const combo of combinationSuggestions) {
    // 【豆包终审新增】年龄限制检查
    if (combo.ageRestriction && ageGroup) {
      const ageNum = parseInt(ageGroup)
      if (ageNum < combo.ageRestriction.min) {
        continue  // 跳过此联用规则，显示年龄限制提示
      }
    }

    let isMatch = true

    for (const [key, expectedValues] of Object.entries(combo.trigger)) {
      const actualValue = answers[key]
      if (!actualValue) {
        isMatch = false
        break
      }
      // expectedValues 可以是数组或单个值
      const expected = Array.isArray(expectedValues) ? expectedValues : [expectedValues]
      const actual = Array.isArray(actualValue) ? actualValue : [actualValue]

      // 只要有交集就匹配
      const hasIntersection = expected.some(ev => actual.includes(ev))
      if (!hasIntersection) {
        isMatch = false
        break
      }
    }

    if (isMatch) {
      matched.push(combo)
    }
  }

  return matched
}
```
